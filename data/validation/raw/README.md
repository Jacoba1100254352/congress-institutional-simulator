# Raw Validation Inputs

Curated, documented empirical extracts live in this directory. Small public API
adapter fixtures belong in `data/validation/fixtures/` and are intentionally
ignored by the raw validation-readiness check.

Current raw samples:

- `bill_progression.csv`: 180-row Congress.gov 118th Congress offset sample,
  generated by `scripts/validation/build_bill_progression_dataset.py`.
- `bill_progression.metadata.md`: generation parameters and sample-level floor,
  committee-report, and enactment rates.
- `voteview_rollcalls.csv`: Voteview 118th Congress member-vote sample for
  party-unity and coalition-size checks.
- `topic_throughput.csv`: topic-level throughput derived from the bill
  progression sample.
- `sponsor_success.csv`: sponsor proposal-access sample with a bounded
  Congress.gov bill-detail enrichment pass.
- `committee_activity.csv`: committee-report signal derived from bill-action
  histories. This is not a hearing-calendar dataset.
- `lobbying_disclosure.csv`: Senate LDA filing activity sample for spending
  concentration and issue-diversity checks.

The current samples are useful for empirical-bridge checks and rough
agenda-access plausibility. They are not complete congressional censuses and
should not be described as fitted empirical validation.
